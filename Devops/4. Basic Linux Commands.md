### **Basic Linux Commands – Interview Notes**

---

## **1. Basic Simple Commands**

* `ls` – list files
* `pwd` – current path
* `whoami` – current user
* `clear` – clear screen

---

## **2. Directory Operations**

* `mkdir` – create directory
* `rmdir` – delete empty directory
* `rm -r` – delete directory
* `tree` – view structure

---

## **3. File Operations**

* `touch` – create file
* `cp` – copy file
* `mv` – move/rename
* `rm` – delete file

---

## **4. Navigate File System**

* `cd /` – root
* `cd ..` – back
* `cd ~` – home

---

## **5. More File & Directory Ops**

* `find` – search files
* `du` – disk usage
* `df` – disk space

---

## **6. More Useful Commands**

* `cat` – view file
* `less` – scroll file
* `head / tail` – file parts
* `grep` – search text

---

## **7. CLI vs GUI**

* **CLI faster & scriptable**
* Used in servers & automation

---

## **8. Display OS Information**

* `uname -a`
* `lsb_release -a`
* `top`

---

## **9. Superuser Commands**

* `sudo`
* `su`

---

### **Interview Questions & Answers**

* **Why CLI over GUI?**
  Faster, automation-friendly.

* **`rm` vs `rm -r`?**
  File vs directory.

* **Purpose of `sudo`?**
  Run admin commands.

---

### **Ultra-Short Summary**

* **Linux = CLI driven**
* **Commands = file, process, system control**
